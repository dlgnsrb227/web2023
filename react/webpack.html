webpack 개념

Webpack은 JavaScript 애플리케이션을 위한 모듈 번들러입니다. 웹팩은 애플리케이션의 코드를 여러 개의 모듈로 분할하고 이를 종속성 그래프로 관리합니다. 그리고 이러한 모듈들을 하나의 파일로 번들링하여 최적화된 형태로 제공합니다.

웹팩의 핵심 개념은 "로더"와 "플러그인"입니다. 로더는 웹팩이 자바스크립트 파일 외에도 CSS, 이미지, 폰트 등 다양한 파일 형식을 이해하고 처리할 수 있게 도와줍니다. 로더는 각 파일을 모듈로 변환하고 웹팩 번들에 포함시킵니다. 플러그인은 로더 이외의 작업을 수행하며, 번들 최적화, 환경 변수 주입, 번들 결과물의 생성 등 다양한 기능을 제공합니다.

npx webpack을 작성하면 파일이 번들링 됩니다. 

const path = require('path');

    module.exports = {
        mode: 'production', //production, development
        entry: {
            index: './src/index.js'
        },
        output: {
            path: path.resolve(__dirname, 'public'),
            filename: 'js/[name].js'
        }
    }

1.7 CSS 번들러 설정

npm install --save-dev style-loader css-loader

const path = require('path');
const HtmlWebpackPlugin = require('html-webpack-plugin');

module.exports = {
    mode: 'production', //production, development
    entry: {
        index: './src/index.js'
    },
    output: {
        path: path.resolve(__dirname, 'public'),
        filename: 'js/[name].js'
    },
    plugins: [new HtmlWebpackPlugin({
        template: './src/index.html',   // 번들 전
        filename: './index.html',       // 번들 후
        hash: true,
        showErrors: true,
    })],
    module: {
        rules: [
            {
                test: /\.css$/i,
                use: ["style-loader", "css-loader"],
            },
        ],
    },
}
